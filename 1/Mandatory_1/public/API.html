API documentation

I've implemented an API to demonstrate how GET, POST, PUT and DELETE works from Postman, cURL or similar

General information

You should use Postman to send API Requests. Find more information under Tools, or google Postman.

Send a GET Request to the following address, to get an updated list of pages and functions

            localhost:8686/api

The following table shows what conventions are relevant, when making an API

As you can see, it should be possible to know what is going on, at a certain link, just from the url itself.

An example: If we send a GET Request to site/users, we would expect to get a list of users

If we send a GET Request to site/users/1, we would expect to get a single user back with the id 1

GET Request

Located at

            localhost:8686/api/writers

POST Request

Located at

            localhost:8686/api/writers

JSON format example

            {
              "fname": "Albert",
              "lname": "camus"
            }

PUT Request

Located at

            localhost:8686/api/writers/[id]

JSON format example

            {
              "fname": "Albert",
              "lname": "Camus"
            }

DELETE Request

Located at

            localhost:8686/api/writers/[id]
